# ATMシステム（Java / JavaFX）

Java と JavaFX を用いて開発した、ATM の基本機能をシミュレートするデスクトップアプリケーションです。  
入金・出金・残高照会といった主要な取引を、シンプルで直感的なUIを通じて体験できるよう設計しています。

---

## 📌 プロジェクト概要

本プロジェクトは、以下の技術の理解と実践を目的として制作しました。

- Javaの主要概念（オブジェクト指向、例外処理、モジュール化設計、カプセル化、データバインディング など）
- JavaFXによるGUI構築
- MVCアーキテクチャの実装
- テキストファイルを用いた基本的なデータ永続化

---

## 🛠 使用技術・ツール

| カテゴリ | 名称 | 理由・役割 |
|---------|------|------------|
| 開発言語 | Java | プラットフォーム非依存・豊富なライブラリ・強力なOOPにより安定した開発が可能 |
| UIフレームワーク | JavaFX | FXMLでUI構造を定義でき、CSSで柔軟なスタイル調整が可能 |
| スタイル設定 | CSS | UIデザインの統一性と保守性向上のため使用 |
| ビルドツール | Gradle | 依存関係管理とビルド自動化を効率化 |
| アーキテクチャ | MVCモデル | 関心の分離により可読性・保守性・再利用性を向上 |
| データ管理 | テキストファイル | 小規模プロジェクトに適したシンプルな永続化手法 |

## 📐 MVCアーキテクチャ構成

### **Model（モデル）**
- 残高データの保持  
- 入出金処理のロジック  
- テキストファイルの読み書き  

### **View（ビュー）**
- JavaFX + FXML + CSS で構築  
- ログイン画面、メインメニュー、各操作画面を表示  
- ユーザー入力を受け取り、Controllerへ通知  

### **Controller（コントローラー）**
- ユーザー操作を受け取り、Modelの処理を実行  
- 処理結果をViewへ反映  
- 画面遷移を管理

## 🎨 UIデザインとCSSの工夫

### **CSSによるスタイル統一**

## 💡 実装機能

### **1. ログイン**
- ユーザー名とPINを入力  
- 正しい場合はメインメニューへ遷移  
- 誤りの場合はエラーメッセージを表示  
- ※デフォルト：ID「shreeram」、PIN「1234」

### **2. 入金**
- 入金額を入力  
- 正の整数か検証  
- 残高を更新し、結果を表示  

### **3. 出金**
- 出金額を入力  
- 残高不足をチェック  
- 問題なければ残高を更新  

### **4. 残高照会**
- 現在の残高を即時表示  

---

## 💾 データ保存（テキストファイル）

### **データ形式**
- `balance.txt` に残高を数値で保存（例：50000）

### **読み込み・書き込み**
- Java標準I/O（Files, BufferedReader/Writer）を使用  
- 入出金後に新しい残高を上書き保存  

### **例外処理**
- IOExceptionなどをtry-catchで処理  
- エラー時はログ出力とユーザー通知  

---

## 🔄 コントローラーの処理フロー（ログイン例）

1. ユーザーがIDとPINを入力  
2. `handleLoginButtonAction()` が呼び出される  
3. 入力値を取得  
4. 認証ロジックで検証  
5. 成功 → メインメニューへ遷移  
6. 失敗 → エラーメッセージ表示
